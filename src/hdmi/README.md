# HDMI出力 ハードウェア設計・ピンアサイン詳細

本ディレクトリ（`src/hdmi`）の実装に対応するFPGAボード、ピン配置、およびHDMI出力（TMDS）の回路設計について記述します。

---

## 1. 対象FPGA仕様

本プロジェクトは **Sipeed Tang Console 138K** 開発ボードをメインターゲットに設計されています。また、macOS環境の制限に対応するためのフォールバック構成もサポートしています。

| 項目 | メインターゲット (通常ビルド) | フォールバックターゲット (macOS) |
|:---|:---|:---|
| **SoM ボード** | Tang Mega 138K | 同左 |
| **FPGA デバイス** | **GW5AST-LV138FPG676AC2/I1** | **GW5AST-LV138PG484AC1/I0** |
| **シリーズ** | Arora V | Arora V |
| **パッケージ** | **FPG676** (BGA 676-pin) | **PG484** (BGA 484-pin) |
| **解説** | 開発ボード上の物理FPGAパッケージ。 | macOS Education版 Gowin EDAでFPG676が未登録の際、フルビルドを通すための仮ターゲット。 |

---

## 2. 詳細ピンアサイン (Physical Pin Mapping)

| 信号名 | 物理ピン (P側) | 物理ピン (N側) | I/O 規格 | バンク | 機能詳細 |
|:---|:---|:---|:---|:---|:---|
| `clk_50m` | **V10** | (単端入力) | LVCMOS33 | Bank 8 | システムクロック (50MHz 水晶) |
| `tmds_d2` (Red) | **AA22** | **AA23** | LVCMOS33D | Bank 3 | TMDS データチャネル 2 |
| `tmds_d1` (Green) | **V24** | **W24** | LVCMOS33D | Bank 3 | TMDS データチャネル 1 |
| `tmds_d0` (Blue) | **AB24** | **AC24** | LVCMOS33D | Bank 3 | TMDS データチャネル 0 (同期信号込) |

| `led` | **U12** | (単端出力) | LVCMOS33 | Bank 10 | ステータスLED (PLLロック時に点灯) |

### 2.2 フォールバックターゲット (PG484 パッケージ)
macOS上で配置配線 (P&R) を完了しビットストリーム (`.fs`) を生成して、書き込みフローを検証するためのピンアサインです。物理的に存在する Bank 3 のI/Oピンから差動ペアを割り当てています。

| `clk_50m` | **V22** | (単端入力) | LVCMOS33 | Bank 8 | システムクロック |
| `tmds_d2` (Red) | **K17** | **J17** | (RTLにて自動) | Bank 3 | 差動出力 |
| `tmds_d1` (Green) | **J15** | **H15** | (RTLにて自動) | Bank 3 | 差動出力 |

| `tmds_clk` | **G15** | **G16** | (RTLにて自動) | Bank 3 | 差動出力 |

> **注意**: PG484でビルドしたビットストリームは、書き込み自体は成功しますが物理的なピン配置が実機開発ボードのHDMIコネクタと異なるため、HDMIモニターへの実際の画面表示は行われません（画面は黒いままとなります）。

## 3. HDMI出力回路設計 (DVI物理層の構成)

HDMI信号は、高速差動デジタル信号である TMDS (Transition Minimized Differential Signaling) を使用してシリアル送信されます。本プロジェクトでは、FPGA内の専用ハードウェアブロック（プリミティブ）を使用して出力回路を構成しています。

### 3.1 送信回路ブロックダイアグラム
```
[Pixel-clk Domain: 25.2MHz]                    [Serial-clk Domain: 126MHz] (DDR: 252Mbps)
 ┌───────────────┐
 │   Encoder     ├─ 10bit TMDS ──► ┌───────────┐
 │ (RGB&Syncを)  │                 │  OSER10   │ ── Serial Bit ──► ┌─────────────┐
 │ (10bitエンコード)│               │(シリアライザ)│                 │ TLVDS_OBUF  │ ──► tmds_p / n
 └───────────────┘                 └───────────┘                 │(差動ドライバ)│
   PCLK: 25.2MHz ───────────────────► PCLK          FCLK: 126MHz ──► └─────────────┘
```

### 3.2 OSER10 (10:1 DDR シリアライザ) プリミティブ
HDMI規格はピクセルクロックの10倍のデータレートを必要とします。640x480@60Hzでは 25.2MHz × 10 = 252 Mbps です。
Gowin FPGAでは、`OSER10` プリミティブを用いて 10:1 のシリアライズを行います。

- **クロック構成**:
  - `PCLK` にピクセルクロック（`25.2 MHz`）を入力します。
  - `FCLK` にシリアル転送用クロック（`126.0 MHz`）を入力します。
  - `OSER10` は内部で **DDR（Double Data Rate）** 動作するため、`FCLK` の立ち上がりと立ち下がりの両方のエッジでデータを1ビットずつシフト出力します。これにより、126MHz × 2 = 252Mbps の転送レートが得られ、ピクセルクロック（25.2MHz）の1周期の間にちょうど10ビットのシリアルデータが出力されます。
- **データ入力**:
  - `D0` 〜 `D9` 入力に、エンコーダが生成した 10-bit の TMDS トークン（`tmds_r`, `tmds_g`, `tmds_b`）をバインドします。

### 3.3 TLVDS_OBUF (差動出力バッファ) プリミティブ
OSER10から出力されたシングルエンド（LVCMOS）のシリアル信号を、HDMIケーブルを通じた伝送に適した差動信号に変換します。

- **動作**:
  - 入力ピン `I` にシリアライザ出力（`ser_d[0-2]`）を入力します。
  - 出力ピン `O`（正相：P側）と `OB`（逆相：N側）から差動信号（`tmds_p`, `tmds_n`）が出力されます。
- **物理規格**:
  - FPGA内部でLVDS差動バッファをアクティブ化し、HDMIの物理レイヤ（TMDS）に適合させます。

### 3.4 ピクセルクロック再生回路 (tmds_clk)
HDMIレシーバー（モニター）は、データ線のサンプリングのためにピクセルクロック信号を同時に受け取る必要があります。データ線と同様の遅延（位相関係）でクロックを送信するため、クロックチャネルもデータチャネルと全く同じシリアライザ経路を経由して出力します。

- **実装**:
  - クロック用の `OSER10` インスタンス（`oser_ck`）の入力ピン `D0` 〜 `D9` に対し、固定パターン **`10'b0000011111`**（D0-D4 = 1, D5-D9 = 0）を入力します。
  - この 10-bit パターンがシリアライズされると、前半5ピクセルクロック期間（FCLKの5周期分）は `High`、後半5ピクセルクロック期間は `Low` となり、デューティ比 50% の `25.2 MHz`（ピクセルクロックと同一周波数）のシリアルクロック信号が再生されます。
  - これをデータチャネル同様に `TLVDS_OBUF` 経由で `tmds_clk_p` / `tmds_clk_n` へと送信することで、データと完全に同期した差動クロック信号を送信します。
