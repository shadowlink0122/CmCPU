# HDMI ASCIIコード表表示設計仕様書

## 1. 概要
本設計書は、HDMIテキスト出力プログラム (`hdmi_text_top`) を、全印刷可能ASCII文字（32〜127）とその文字コードをグリッド形式で一覧表示する「ASCIIコード表表示プログラム」へと変更するための設計について定めます。

文字をより多く配置するため、表示領域を従来のGameBoy Color解像度（160x144, 3倍スケール）から、HDMIの標準有効画素領域である **640x480 (等倍スケール)** へと拡張し、8x8フォント時に最大 **80列 × 60行 (4800文字)** の表示を可能にします。

## 2. 設計方針

### 2.1 画面解像度とスケール設定の変更
`src/hdmi/text/gbc_display.cm` の定数を以下のように変更し、640x480領域全体を等倍で logical 表示領域とします：
- `SCALE` : 1 (等倍)
- `H_OFFSET` : 0 (黒帯なし)
- `V_OFFSET` : 0 (黒帯なし)
- `DISP_WIDTH` : 640
- `DISP_HEIGHT` : 480

等倍への変更に伴い、論理座標 `gbc_x` (最大639) と `gbc_y` (最大479) が `utiny` (8ビット) の表現範囲を超えてオーバーフローするため、型を `ushort` (16ビット無符号整数) へと拡張します。

### 2.2 テキストレンダラーの一般化とバッファ拡張
- `src/hdmi/text/text_renderer.cm` 内の `text_buf` のサイズを 360 から **4800** に拡張します。
- `gbc_x` および `gbc_y` の型変更に伴い、レンダラー側の入力受取部分の型を `ushort` に適応させます。

### 2.3 ASCIIコード表のレイアウト設計
画面解像度が 80列 × 60行 に拡張されるため、画面中央付近に美しく整形されたテーブルを表示します。

#### 2.3.1 グリッドレイアウト
ASCII文字 32〜127 (合計96文字) を 6列 × 16行 のマトリクスで配置します：
- **列形式**: ` Dec Ch |` （例: ` 033 [!] |`）
  - 各列の幅: 10文字 (文字情報7文字 + スペース1文字 + セパレータ `|` 1文字 + スペース1文字)
  - 6列合計幅: 60文字 (最後の列にはセパレータなし)
- **配置位置**:
  - 水平方向: 10列目から開始 (左右にそれぞれ10マスの余白を空けてセンタリング)
  - 垂直方向: 15行目から30行目までの16行

#### 2.3.2 ヘッダー情報
- **タイトル**: 8行目中央付近に `=== ASCII Code Table ===` を配置。
- **カラムヘッダー**: 12行目に列の見出し `  Dec Ch  |  Dec Ch  | ... ` を配置。

### 2.4 アニメーションコントローラの変更 (`animation_ctrl.cm`)
起動時にバッファをスペース `' '` でクリアし、その後一定フレーム間隔（例: 10フレームごと）で以下の行順にテキストを描画するステートマシンを実装します：
- `state = 0` : 起動待機 (60フレーム)
- `state = 1` (描画処理中): `row_idx` カウンタを 0 から 18 まで進める：
  - `row_idx == 0` : メインタイトルを出力
  - `row_idx == 1` : カラムヘッダーを出力
  - `row_idx >= 2` かつ `row_idx <= 17` : 各行 $r = row\_idx - 2$ の6列分データを出力
- `state = 2` : 描画完了（DONE、カーソル点滅などの状態表示）

### 2.5 数値のデシマル文字列変換ロジック
Cm言語には標準で浮動小数点や複雑なフォーマット出力がないため、Cm内の演算で直接桁ごとのASCIIコードに分解します：
- 100の位: `(val / 100) + '0'`
- 10の位: `((val / 10) % 10) + '0'`
- 1の位: `(val % 10) + '0'`

---

## 3. 影響範囲
本変更はテキスト表示回路 (`hdmi_text_top`) にのみ適用され、カラーバー表示 (`main.cm`) には影響を及ぼしません。
また、Pythonの自動ジェネレータ `generate_font.py` も画面解像度 640x480 に基づく列数/バッファサイズ（80x60 = 4800）をエクスポートするように整合性を合わせます。
