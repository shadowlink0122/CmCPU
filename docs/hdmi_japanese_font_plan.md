# HDMI 日本語（ひらがな・カタカナ）フォント設計書

本ドキュメントは、HDMI表示モジュールにおける日本語（ひらがな・カタカナ）表示機能の設計およびマッピング対応表を定義します。

## 背景と目的
0〜32番の文字コード領域は制御文字および標準スペース用として予約し、日本語フォントを配置しません。
標準ASCIIコード（33〜127）の直後である「128番」から、ひらがな・カタカナ・各種記号を連続したインデックスでマッピングします。
1バイト（128〜255の128スロット）に収めるため、ひらがな・カタカナは基本文字＋独立した濁点・半濁点（結合文字方式）として定義します。これにより、すべての必要な文字（計118文字）を128〜245番に美しく連続して配置可能となります。

## フォントソース
- 8x8ビットマップフォントとして定評のある「美咲フォント（Misaki Gothic）」の BDF 配布ファイル（`misaki_gothic.bdf`）を使用します。

## 文字コードマッピング対応表 (0〜255)

### 1. 制御文字・スペース領域 (0〜32)
標準制御文字およびスペース領域として予約します。日本語フォントは割り当てません。

### 2. ASCII 印刷可能文字領域 (33〜127)
標準ASCIIコード表に準拠します。
- 34: `"`, 126: `~`, 127: DEL

### 3. 日本語ひらがな領域 (128〜182)
- **128〜132**: あ, い, う, え, お
- **133〜137**: か, き, く, け, こ
- **138〜142**: さ, し, す, せ, そ
- **143〜147**: た, ち, つ, て, と
- **148〜152**: な, に, ぬ, ね, の
- **153〜157**: は, ひ, ふ, へ, ほ
- **158〜162**: ま, み, む, め, も
- **163〜165**: や, ゆ, よ
- **166〜170**: ら, り, る, れ, ろ
- **171〜173**: わ, を, ん
- **174〜178**: ぁ, ぃ, ぅ, ぇ, ぉ
- **179**: っ
- **180〜182**: ゃ, ゅ, ょ

### 4. 日本語カタカナ領域 (183〜237)
- **183〜187**: ア, イ, ウ, エ, オ
- **188〜192**: カ, キ, ク, ケ, コ
- **193〜197**: サ, シ, ス, セ, ソ
- **198〜202**: タ, チ, ツ, テ, ト
- **203〜207**: ナ, ニ, ヌ, ネ, ノ
- **208〜212**: ハ, ヒ, フ, ヘ, ホ
- **213〜217**: マ, ミ, ム, メ, モ
- **218〜220**: ヤ, ユ, ヨ
- **221〜225**: ラ, リ, ル, レ, ロ
- **226〜228**: ワ, ヲ, ン
- **229〜233**: ァ, ィ, ゥ, ェ, ォ
- **234**: ッ
- **235〜237**: ャ, ュ, ョ

### 5. 日本語記号領域 (238〜245)
- **238**: 、 (読点)
- **239**: 。 (句点)
- **240**: 「 (鍵括弧始)
- **241**: 」 (鍵括弧終)
- **242**: ー (長音記号)
- **243**: ・ (中黒)
- **244**: ゛ (濁点)
- **245**: ゜ (半濁点)

## 実装および実行プロセス
1. `.tmp/download_and_add_jp.py` のマッピング対応表を更新し、実行して `font_rom.txt` を再生成する。
2. `generate_font.py` で `font_rom.cm` を再生成する。
3. `animation_ctrl.cm` の描画ロジックを更新する。
4. ビルド・合成・書き込み・確認を行う。
