# HDMI 日本語（ひらがな・カタカナ）フォント設計書

本ドキュメントは、HDMI表示モジュールにおける日本語（ひらがな・カタカナ）表示機能の設計およびマッピング対応表を定義します。

## 背景と目的
ゲームボーイ等のレトロゲーム機では、限られたメモリ空間で文字を表示するため、独自のキャラクターマッピング（1バイトで表されるコード）や precomposed（濁点等があらかじめ合成された）グリフを採用しています。
本プロジェクトでも、標準ASCIIコード（32〜127）を完全に維持しつつ、残りの未使用領域である「1〜31」および「128〜255」の計159スロットを利用して、日本語ひらがな・カタカナ・各種記号をマッピングし、実機での表示を可能にします。

## フォントソース
- 8x8ビットマップフォントとして定評のある「美咲フォント（Misaki Gothic）」の BDF 配布ファイル（`misaki_gothic.bdf`）を使用します。
- このフォントは、8x8サイズの限られた解像度の中で非常に美しく日本語を描画できるよう設計されています。

## 除外文字の選定
1バイト（256個の文字空間）内にASCII（96文字）と日本語文字を共存させるため、全日本語文字（176文字）から使用頻度の極めて低い以下の17文字を除外します。これにより、すべての文字が結合処理なし（precomposed）で1バイトに収まります。
1. ひらがな・カタカナの「ゐ、ゑ、ゎ、ヰ、ヱ、ヮ」 (6文字)
2. ほとんど使われない濁点付きの「ぢ、づ、ヂ、ヅ」 (4文字)
3. 外来語用の特殊文字「ゔ、ヴ」 (2文字)
4. ひらがなの小文字「ぁ、ぃ、ぅ、ぇ、ぉ」 (5文字) (カタカナの小文字「ァ、ィ、ゥ、ェ、ォ」は外来語等の表記に必須なため残します)

## 文字コードマッピング対応表 (0〜255)

### 1. ASCII 領域 (32〜127)
標準ASCIIコード表に準拠します。
- 32: Space, 34: `"`, 126: `~`, 127: Space/DEL

### 2. 日本語ひらがな領域 (1〜31, 128〜169)
- **1〜5**: あ, い, う, え, お
- **6〜15**: か, が, き, ぎ, く, ぐ, け, げ, こ, ご
- **16〜25**: さ, ざ, し, じ, す, ず, せ, ぜ, そ, ぞ
- **26〜31**: た, だ, ち, っ, つ, て
- **128〜129**: で, と
- **130〜135**: ど, な, に, ぬ, ね, の
- **136〜140**: は, ば, ぱ, ひ, び
- **141〜143**: ぴ, ふ, ぶ
- **144〜147**: ぷ, へ, べ, ぺ
- **148〜150**: ほ, ぼ, ぽ
- **151〜155**: ま, み, む, め, も
- **156〜158**: や, ゆ, よ
- **159〜163**: ら, り, る, れ, ろ
- **164〜166**: わ, を, ん
- **167〜169**: ゃ, ゅ, ょ

### 3. 日本語カタカナ領域 (170〜247)
- **170〜174**: ア, イ, ウ, エ, オ
- **175〜184**: カ, ガ, キ, ギ, ク, グ, ケ, ゲ, コ, ゴ
- **185〜194**: サ, ザ, シ, ジ, ス, ズ, セ, ゼ, ソ, ゾ
- **195〜203**: タ, ダ, チ, ッ, ツ, テ, デ, ト, ド
- **204〜208**: ナ, ニ, ヌ, ネ, ノ
- **209〜217**: ハ, バ, パ, ヒ, ビ, ピ, フ, ブ, プ
- **218〜223**: ヘ, ベ, ペ, ホ, ボ, ポ
- **224〜228**: マ, ミ, ム, メ, モ
- **229〜231**: ヤ, ユ, ヨ
- **232〜236**: ラ, リ, ル, レ, ロ
- **237〜239**: ワ, ヲ, ン
- **240〜244**: ァ, ィ, ゥ, ェ, ォ
- **245〜247**: ャ, ュ, ョ

### 4. 日本語記号領域 (248〜255)
- **248**: 、 (読点)
- **249**: 。 (句点)
- **250**: 「 (鍵括弧始)
- **251**: 」 (鍵括弧終)
- **252**: ー (長音記号)
- **253**: ・ (中黒)
- **254**: ゛ (濁点)
- **255**: ゜ (半濁点)

## 実装および実行プロセス
1. `font_rom.txt` 内の `char 99` （小文字 c）の先端ビットの不具合を修正する。
2. BDFファイルのパースおよび `font_rom.txt` への追記を行う Python スクリプトを実行する。
3. `generate_font.py` で `font_rom.cm` を再生成する。
4. `animation_ctrl.cm` の描画ロジックを修正して、16x16または8x32等の全文字表示グリッドを出力するよう更新する。
5. ビルド・合成・書き込み・確認を行う。
