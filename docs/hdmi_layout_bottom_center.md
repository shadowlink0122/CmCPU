# HDMIレイアウト変更設計書（中央下揃え配置）

## 1. 概要
ASCIIコード表の表示位置について、ユーザーからの要望に基づき「中央下揃え（bottom-center）」になるように配置を修正します。

## 2. 画面領域と現在の配置
* **全体の画面解像度**: 640 × 480 ピクセル
* **フォントサイズ**: 8 × 8 ピクセル
* **テキストグリッドサイズ**: 横 80 文字 (TEXT_COLS) × 縦 60 行 (TEXT_ROWS)

### 2.1 変更前の配置
* **ヘッダー**: 8行目 (中央配置)
* **カラム見出し**: 12行目 (中央配置)
* **グリッドデータ**: 15行目〜30行目 (中央配置)
* **下部余白**: 31行目〜59行目 (29行分が空き領域)

横方向（カラム数）はすでに左右対称（余白10文字）で中央配置されているため、縦方向（行数）を下に移動することで「中央下揃え」を実現します。

---

## 3. 変更後の配置設計（下揃え）
画面の最下部（59行目）の直前まで表が収まるように、以下のように行配置（オフセット）を下方向へシフトします。最下部の59行目は1行分だけマージン（余白）として残します。

* **グリッドデータ最終行**: 58行目
* **グリッドデータ開始行**: 43行目 (16行分: 43〜58行目)
* **カラム見出し行**: 40行目 (グリッド開始の3行上、余白2行)
* **ヘッダー行**: 36行目 (カラム見出しの4行上、余白3行)

### 配置まとめ表
| 要素 | 変更前の行番号 | 変更後の行番号 | 横位置（カラム） |
| :--- | :---: | :---: | :--- |
| ヘッダー (`Header`) | 8 | **36** | 28 (中央揃え) |
| カラム見出し (`ColHeader`) | 12 | **40** | 10 (中央揃え) |
| グリッドデータ開始 (`Row 0`) | 15 | **43** | 10 (中央揃え) |
| グリッドデータ終了 (`Row 15`) | 30 | **58** | 10 (中央揃え) |

---

## 4. 実装変更点

`src/hdmi/text/animation_ctrl.cm` の描画先頭アドレス計算部分を変更します。

### 4.1 ヘッダーのアドレス計算
```cm
// 変更前 (Row 8)
text_addr = (8 * (TEXT_COLS as ushort) + 28 + (write_cnt as ushort)) as ushort;

// 変更後 (Row 36)
text_addr = (36 * (TEXT_COLS as ushort) + 28 + (write_cnt as ushort)) as ushort;
```

### 4.2 カラム見出しのアドレス計算
```cm
// 変更前 (Row 12)
text_addr = (12 * (TEXT_COLS as ushort) + 10 + (write_cnt as ushort)) as ushort;

// 変更後 (Row 40)
text_addr = (40 * (TEXT_COLS as ushort) + 10 + (write_cnt as ushort)) as ushort;
```

### 4.3 グリッドデータ行のアドレス計算
```cm
// 変更前 (Row 15 + r)
text_addr = (((15 + r as ushort) * (TEXT_COLS as ushort)) + 10 + (write_cnt as ushort)) as ushort;

// 変更後 (Row 43 + r)
text_addr = (((43 + r as ushort) * (TEXT_COLS as ushort)) + 10 + (write_cnt as ushort)) as ushort;
```
