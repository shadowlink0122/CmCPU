# HDMI 汎用設計およびピン制約復元設計書

## 1. 汎用的なHDMI出力（映像のみ）のモジュールアーキテクチャ

HDMI出力を特定のプログラムに依存しない汎用的なコンポーネントとするため、以下のような「送信基盤（HDMI Core）」と「コンテンツ生成（Video Generator）」を完全に分離した疎結合設計を適用します。

### 1.1 HDMI コア (共通ライブラリ群)
以下のモジュールは、HDMI信号の生成・伝送にのみ特化し、特定の画素パターンには依存しません。
- **`pll.cm`**: 外部クロックからピクセルクロック（25.2MHz）とシリアルシリアライザ用高速クロック（126MHz）を生成。
- **`timing.cm`**: 水平/垂直アクティブ領域・フロントポーチ・バックポーチ・同期信号期間のカウンタ（`hc`, `vc`）およびデータイネーブル信号（`de_reg`）の生成。共通画素値変数（`r_out`, `g_out`, `b_out`）の定義。
- **`encoder.cm`**: RGB各8ビットデータ（`r_out`, `g_out`, `b_out`）を受け取り、DVI/TMDS 10-bitパラレルコードに連続代入（`assign`）を介して遅延なくエンコード。
- **`oser.cm` & `obuf.cm`**: パラレル信号を10:1 DDRシリアライズし、TLVDS差動信号に変換して物理ピンから出力。

### 1.2 コンテンツ生成モジュール (個別差分)
任意の表示プログラムは、共通の `timing.cm` が生成する `hc`/`vc` 座標情報および `de_reg` 信号に同期して、`r_out`, `g_out`, `b_out` に画素値を書き込むだけで動作します。
- **`pattern.cm`**: 画面を縦に8分割し、カラーバーパターンを生成。
- **`text_renderer.cm`**: フォントデータをインラインで読み出し、Hello World! テキストと点滅カーソルを生成。

## 2. ピン制約とLED制御の復元

各プログラム固有の表示要件とボードのI/Oピン配置を合わせるため、以下の通りピン制約ファイル（`.cst`）とトップレベルのポート定義を分離します。

### 2.1 カラーバー (`main.cm` + `tang_console_138k_hdmi.cst`)
- 出力ポート: `led` 1ポート
- 割り当て: PLLロック状態表示
- 制約ファイル: `tang_console_138k_hdmi.cst`

### 2.2 テキスト・アニメーション (`hdmi_text_top.cm` + `tang_console_138k_hdmi_text.cst`)
- 出力ポート: `led_ready` (U12) および `led_done` (G11)
- 割り当て: `led_ready` はタイピングアニメーション中、`led_done` はタイピング完了時。
- 制約ファイル: `tang_console_138k_hdmi_text.cst`

これにより、共通のHDMI映像伝送路は全く変更することなく、各プロジェクトは独自のピン定義および制御ロジックを独立して保持できます。
