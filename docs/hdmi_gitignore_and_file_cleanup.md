# 生成後 SV ファイルの Git 管理除外および不要ファイルのクリーンアップ

## 1. 背景と目的

CmCPU プロジェクトでは、Cm コンパイラによって SystemVerilog（`*.sv`）コードがビルド時に自動生成されるため、これらの生成後ファイルを Git リポジトリの管理下から排除し、誤ったコミットを防止する必要があります。

また、以前の動作確認用あるいは移植元として使用されていた手動作成の SV ファイルや、不要になった中間スクリプト等を整理し、実装ディレクトリ（`src/`）をクリーンな状態に保ちます。

## 2. 実施内容

### 2.1 .gitignore の更新
プロジェクトルートの `.gitignore` に以下のルールを追加し、SystemVerilog ファイルがコミット対象に含まれないようにします。

```text
# 生成後 SystemVerilog ファイル
*.sv
```

### 2.2 不要ファイルの削除
移植元であり現在は Cm コンパイラでの生成コード（`src/hdmi/main.cm` 等）に置き換わったため不要となった以下のファイルを削除します。

- `src/hdmi/hdmi_colorbar_top.sv`

`src` ディレクトリ内の他のファイルは、稼働中のソースコード（`.cm`）、制約ファイル（`.cst`）、および Gowin ビルド用のTclスクリプト（`.tcl`）のみであることを確認済みであり、これら以外に不要なファイルはありません。
