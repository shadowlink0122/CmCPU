# HDMI出力停止問題の調査とCFG探索・グローバル変数重複排除の再適用計画

## 1. 現象と原因
CmコンパイラのSystemVerilogバックエンドコード（`codegen.cpp`）をコミット `418d631` にリバートした結果、HDMI出力が全くされない現象が発生しました。

生成された `build/hdmi/hdmi_colorbar.sv` のTMDSエンコーダ部を確認したところ、以下の現象が発生していることが確認されました。
- Rチャネルのエンコード処理の内部（特定のif/else分岐内）にGチャネルのエンコード処理がネストされ、さらにその内部にBチャネルのエンコード処理がネストされている。
- 本来は並列（シーケンシャル）に実行されるべきR/G/Bのエンコード処理が、深いネスト構造になってしまっている。

この原因は、リバートによってコミット `72a4c8f` （`findMergeBlock` のelseブランチCFG探索における `SwitchInt`/`Call` 追跡の追加）が失われたためです。これにより、ネストされた `if-else` 構造を持つCFGで正しい合流ブロックが検出できなくなり、後続のコードが直前の `if` ブロック内部に誤ってネストされてしまいました。

また、多重インポートによるグローバル変数の重複定義バグを解消するためのコミット `8e9f13b` もリバートで失われているため、これも合わせて再適用する必要があります。

## 2. 対応方針
Cmコンパイラ（`Cm/src/codegen/sv/codegen.cpp`）に対し、失われた以下の2つの修正を再適用します。

### (1) `findMergeBlock` のelseブランチCFG探索修正 (`72a4c8f` の再適用)
`findMergeBlock` のelseブランチ探索ループにおいて、 `Goto` だけでなく `SwitchInt` および `Call` ターミネータの後続ブロックも追跡するように修正します。これにより、R/G/Bエンコード部のようなネストされた制御フローでの合流ブロック検出を正常化します。

### (2) グローバル変数/ポートの重複排除 (`8e9f13b` の再適用)
多重インポートされた同一のグローバル変数やポートの宣言が重複して出力されるのを防ぐため、 `SVCodeGen::analyzeMIR` に `emitted_var_names` （`std::set<std::string>`）を導入し、同一フラット名（名前空間 `::` を除去した名前）の変数の複数回宣言を防止します。

## 3. 動作検証
1. Cmコンパイラのビルドが正常に通ることを確認（`make build`）
2. SystemVerilogテストスイートでデグレーションがないことを確認（`make tsv`）
3. `builder.sh hdmi` を実行し、生成される `hdmi_colorbar.sv` のTMDSエンコーダ部のネストが解消され、元の並列な構造に復元していることを確認
4. `./builder.sh --apply hdmi` による実機書き込みを行い、HDMI出力（カラーバー）が正しく画面に表示されることを確認
