# HDMIフォントグリフ縦整列（下揃え・中央揃え）設計書

## 1. 概要
`font_rom.txt` に定義されている一部のフォントグリフ（小文字、カンマやピリオドなどの記号）が、セルの上部（Row 0）に寄って定義されているため、画面表示時に文字のベースラインが揃わず視覚的に美しくありませんでした。
本ドキュメントでは、これらの文字グリフをセル（8x8）内で「中央下揃え（ベースライン Row 6 揃え）」にするための修正設計について記述します。

## 2. 課題と設計

### 2.1 小文字のアラインメント（Row 6 基準）
アセンダのない標準的な小文字は、底辺が Row 6（0-indexed）に揃うのが理想的です。
元データでは、小文字（`a`, `c`, `e` など）は Row 0 から Row 4（5行）に定義されていたため、下に2行シフトして Row 2 から Row 6 に配置します。
対象文字: `a, c, e, m, n, o, r, s, u, v, w, x, z` (2行シフト)

### 2.2 ディセンダ付き小文字のアラインメント（Row 7 基準）
ディセンダ（下にはみ出る部分）を持つ小文字（`g`, `j`, `p`, `q`, `y`）は、底辺がセルの最下部 Row 7 に揃うように配置します。
- `g, j, p, q` (元データ Row 0〜5 の6行): 下に2行シフトして Row 2〜7 に配置。
- `y` (元データ Row 0〜4 の5行): 下に3行シフトして Row 3〜7 に配置。

### 2.3 記号類のアラインメント
各記号の役割に応じて、適切な縦位置に揃うように下にシフトします。
- **ハイフン (`-`)**: Row 0 ⇒ Row 3 (3行シフト、上下中央)
- **チルダ (`~`)**: Row 0〜1 ⇒ Row 2〜3 (2行シフト)
- **カンマ (`,`)**: Row 0〜2 ⇒ Row 5〜7 (5行シフト、ディセンダ位置)
- **セミコロン (`;`)**: Row 0〜5 ⇒ Row 2〜7 (2行シフト、ディセンダ位置)
- **ピリオド (`.`)**: Row 0〜1 ⇒ Row 5〜6 (5行シフト、ベースライン上)
- **コロン (`:`)**: Row 0〜4 ⇒ Row 2〜6 (2行シフト)
- **プラス (`+`)**: Row 0〜4 ⇒ Row 2〜6 (2行シフト)
- **イコール (`=`)**: Row 0〜2 ⇒ Row 2〜4 (2行シフト)

---

## 3. 実装と自動生成フロー

1. `font_rom.txt` に定義された該当する文字のドットマトリクスを、上記設計値に基づいて行シフト（上部に `........` を挿入し、下部をカット）して上書き保存します。
2. `python3 src/hdmi/text/generate_font.py` を実行し、修正された `font_rom.txt` から SystemVerilog 生成用の `font_rom.cm` を再生成します。
3. HDMI表示用の画面レイアウト（ヘッダー行: 8、見出し行: 12、データ開始行: 15）は変更せず、そのままの配置で描画します。
